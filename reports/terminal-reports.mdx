The Terminal Reports section in Lango gives bank operations teams complete visibility over their POS device estate. You can track the total number of terminals registered in the system, review which devices are currently assigned to merchants, identify terminals that have been returned or decommissioned, and monitor real-time device health through the Terminal Status report. Together these reports support fleet planning, capacity reviews, SLA compliance, and device lifecycle management — all exportable in CSV, Excel, or PDF.

<Note>
  Access to terminal reports requires the `lite.reports.view` permission. Contact your system administrator if the Reports menu is not visible in your sidebar.
</Note>

## Navigating to Terminal Reports

In the left-hand sidebar, click **Reports** to expand the reports menu. Terminal report sub-pages are listed under this section. You can navigate directly to `/reports/terminals` and select the specific report from the sub-navigation, or click each report link directly from the sidebar.

## Total Terminals Report

The **Total Terminals** report (`/reports/terminals/total-terminals`) provides a complete inventory of every terminal registered in Lango, regardless of its current deployment status. Use this report to answer questions such as "How many devices has the bank deployed in a given period?" or "Which branch holds the highest concentration of devices?"

### Columns

| Column | Description |
|---|---|
| **Date Deployed** | The delivery date when the terminal was first recorded in the system. |
| **Merchant Name** | The name of the merchant the terminal is associated with. |
| **Terminal No.** | The terminal's serial number. |
| **Merchant ID (MID)** | The Merchant ID assigned to the device profile. |
| **Business Type** | The company type of the merchant (e.g., Sole Proprietor, Limited Company). |
| **Branch** | The bank branch linked to the merchant. |
| **Bank Region** | The bank region associated with the merchant's branch. |
| **Country Region** | The geographic zone of the merchant's branch. |

## Assigned Terminals Report

The **Assigned Terminals** report (`/reports/terminals/assigned-terminals`) lists all terminals currently in an **ASSIGNED** state — meaning they have been formally deployed to a merchant. This report includes assignment metadata that the Total Terminals report does not, making it the right tool for tracking when devices were handed over and by whom.

### Columns

| Column | Description |
|---|---|
| **Date Created** | The delivery date when the terminal was first recorded in the system. |
| **Application Type** | Whether the assignment was for a **New** merchant or a terminal **Replacement**. |
| **Merchant ID (MID)** | The Merchant ID assigned to the device profile. |
| **Merchant Name** | The name of the merchant the terminal is assigned to. |
| **Customer Type** | The customer type classification of the assigned merchant. |
| **Terminal Serial Number** | The device's serial number. |
| **TID** | The terminal identifier assigned to the device profile. |
| **Date Assigned** | The timestamp when the terminal was formally assigned to the merchant. |
| **Assigned By** | The portal username of the operator who performed the assignment. |
| **Approved By** | The portal username of the operator who approved the assignment. |

## Returned / Closed Terminals Report

The **Returned / Closed Terminals** report (`/reports/terminals/returned-closed-terminals`) surfaces terminals in a **DECOMMISSION** or **INREPAIR** status — devices that have been pulled from service and are either awaiting repair or permanently retired. Use this report to manage device retrieval workflows and track your fleet's attrition rate.

The report includes the same columns as the Total Terminals report, with the **Date Returned/Closed** replacing the deployment date to reflect when the terminal was last updated to a closed state.

<Tip>
  Review the Returned / Closed report monthly to identify devices in **INREPAIR** status that have exceeded your expected repair turnaround time — these may need to be escalated or written off.
</Tip>

## Terminal Status Report

The **Terminal Status** report (`/reports/terminals/terminal-status-reports`) gives you a real-time health snapshot of every terminal in the estate. It is the most comprehensive terminal report for day-to-day operational monitoring, combining device status, merchant context, and communication timestamps in a single view.

### Status Values

| Status | Meaning |
|---|---|
| **ACTIVE** | The terminal is processing transactions normally. |
| **IDLE** | The terminal is assigned but has not transacted recently. |
| **INACTIVE** | The terminal is not communicating with the platform. |
| **TMS_INACTIVE** | The terminal has been flagged as inactive by the Terminal Management System. |
| **PENDING_ACTIVATION** | The terminal has been assigned but not yet activated. |

### Columns

| Column | Description |
|---|---|
| **Terminal Status** | The current operational status of the device. |
| **Merchant Name** | The merchant the terminal is assigned to. |
| **Terminal Serial Number** | The device's serial number. |
| **Merchant ID (MID)** | The Merchant ID on the device profile. |
| **TID** | The terminal identifier. |
| **Region Name** | The bank region linked to the merchant's branch. |
| **Status** | The terminal's lifecycle status: ASSIGNED, DECOMMISSION, INREPAIR, or INSTORE. |
| **Branch Name** | The bank branch linked to the merchant. |
| **Last Communication Timestamp** | The most recent time the device contacted the platform. |
| **Last Transaction Timestamp** | The timestamp of the most recent transaction processed by the terminal. |

## Filtering Terminal Reports

All terminal reports share a common set of filter fields available in the search panel above the table:

- **Date Range** — Filter by deployment date, assignment date, or last communication timestamp depending on the report.
- **Terminal Serial Number** — Look up a specific device.
- **Terminal Status** — Filter to ACTIVE, IDLE, INACTIVE, TMS\_INACTIVE, or PENDING\_ACTIVATION.
- **Lifecycle Status** — Filter to ASSIGNED, DECOMMISSION, INREPAIR, or INSTORE.
- **Merchant Name** — Search by merchant name using the typeahead selector.
- **Region** — Filter to a specific bank region.
- **Branch** — Filter to a specific bank branch.

Apply one or more filters and click **Search** to refresh the table.

<Note>
  Combine the **Terminal Status** report with the **Transaction Reports** to identify high-volume devices (many transactions, ACTIVE) versus dormant devices (few or no transactions, IDLE or INACTIVE) across your entire estate.
</Note>

## Exporting Terminal Reports

All terminal reports support row-level selection and three export formats. Select specific rows to limit the export, or leave all rows unchecked to export the full filtered result set.

<Steps>
  <Step title="Open the relevant terminal report">
    Navigate to the Total Terminals, Assigned Terminals, Returned/Closed, or Terminal Status report from the Reports sidebar.
  </Step>
  <Step title="Apply filters">
    Set a date range and any additional filters (status, merchant, branch, region) to scope your results. Click **Search**.
  </Step>
  <Step title="Select rows (optional)">
    Check individual rows to export a specific subset, or leave all unchecked to export the full filtered view.
  </Step>
  <Step title="Click your preferred export format">
    In the top-right toolbar, click:
    - **Export to CSV** — Comma-separated file for data tools and spreadsheets.
    - **Export to Excel** — `.xlsx` file for review in Microsoft Excel.
    - **Export to PDF** — Formatted document for printing or compliance submission.
  </Step>
  <Step title="Download the file">
    The browser downloads the file immediately. Filenames follow the pattern `Total_Terminals_Reports.csv`, `Terminals_Status_Reports.pdf`, etc.
  </Step>
</Steps>

<Tip>
  Use the **Inactive Terminals** filter in the Terminal Status report (set Terminal Status to **INACTIVE** or **TMS\_INACTIVE**) to build a retrieval list. Export to Excel, sort by **Last Transaction Timestamp**, and prioritize the devices that have been silent the longest for field collection.
</Tip>
