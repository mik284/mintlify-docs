The Inventory module is your centralised register of every physical POS terminal your bank owns or manages. Whether a device is sitting in the warehouse, deployed at a merchant outlet, or temporarily pulled for repair, the Inventory gives you a single place to find it, inspect its details, and manage its assignment. Every terminal in the system — regardless of its current status — appears here alongside its hardware identifiers and deployment information.

## Navigating to Inventory

To open the terminal inventory, select **Inventory** from the left navigation sidebar, then click **Terminals**. The portal routes you to `/inventory/terminals`, where the full terminal list loads automatically.

<Note>
  You need both the `canViewInventory` and `canViewTerminal` permissions to access the Inventory module. If either permission is missing from your role, contact your system administrator.
</Note>

## Terminal List

The terminal list displays all devices registered in your estate. Each row represents a single physical terminal and shows the following columns:

| Column | Description |
|---|---|
| **Terminal Serial Number** | The unique hardware serial number printed on the device. Click the serial number to open the terminal's full detail page. |
| **Terminal Model** | The terminal model name (for example, *PAX A920* or *Ingenico Move 5000*), derived from the device's registered model in Master Data. |
| **MID** | The Merchant ID assigned through the terminal's device profile. Displays a dash if no device profile is linked. |
| **Customer Name** | The merchant customer the terminal is currently assigned to. Unassigned terminals show no value in this column. |
| **Terminal Outlet** | The merchant outlet where the terminal is deployed. Populated once the terminal is assigned to an outlet via a device profile. |
| **Terminal Profile** | The device profile currently linked to the terminal, which governs its parameter settings and operator access. |
| **Terminal Status** | The current lifecycle state of the terminal. See [Terminal Status Values](#terminal-status-values) below. |
| **Terminal State** | The real-time connectivity state of the terminal (for example, **ACTIVE** or **INACTIVE**). Populated only for terminals with a **Terminal Status** of **ASSIGNED**. |

### Terminal Status Values

A terminal's **Terminal Status** reflects where it is in its lifecycle:

| Status | Meaning |
|---|---|
| **INSTORE** | The terminal is in stock and has not yet been deployed to a customer or outlet. |
| **ASSIGNED** | The terminal has been assigned to a customer outlet and is actively in the field. |
| **INREPAIR** | The terminal has been temporarily pulled from service for repair or inspection. |
| **DECOMMISSION** | The terminal has been written off and is no longer in active use. A decommissioned terminal displays a prominent alert banner on its detail page. |

## Viewing Terminal Details

Clicking any terminal's serial number in the list opens its detail page at `/inventory/terminals/:id`. The detail page is organised into tabs that give you a complete picture of the device:

<Tabs>
  <Tab title="Dashboard">
    Shows hardware details (serial number, IMEI, part number, tracking number, model image), the assigned customer and outlet, the linked device profile, last transaction time, last communication snapshot, and real-time terminal state.
  </Tab>
  <Tab title="Transactions">
    Lists all transactions processed through this terminal. Requires the `canViewTransaction` permission to appear.
  </Tab>
  <Tab title="Operators">
    Shows operators registered to this terminal via its device profile. Requires the `canViewOperator` permission to appear.
  </Tab>
  <Tab title="History">
    Displays a full audit trail of every action taken on this terminal — assignments, updates, status changes — including the user responsible and the timestamp.
  </Tab>
</Tabs>

## Assigning a Terminal

When a new terminal arrives in your stock, it appears in the inventory with a status of **INSTORE**. Use the assignment flow to link it to a customer outlet and make it operational.

<Steps>
  <Step title="Locate the terminal">
    From the terminal list at `/inventory/terminals`, find the terminal you want to assign. You can search by serial number or filter by status, model, or customer name to narrow results.
  </Step>
  <Step title="Open the terminal record">
    Click the terminal's serial number to open its detail page, or select the terminal using the row checkbox to expose bulk action controls in the toolbar.
  </Step>
  <Step title="Initiate assignment">
    In the terminal detail page or from the action dropdown in the toolbar, click **Assign**. A modal dialog opens prompting you to select the destination.
  </Step>
  <Step title="Select customer and outlet">
    Choose the target customer from the dropdown, then select the specific outlet where the terminal will be deployed. Outlets are filtered to match the selected customer.
  </Step>
  <Step title="Select a device profile">
    Choose the device profile that will govern this terminal's behaviour at the outlet. Device profiles define the parameter set applied to the terminal.
  </Step>
  <Step title="Confirm assignment">
    Review your selections and click **Confirm**. The terminal status updates to **ASSIGNED** and the customer and outlet columns in the inventory list reflect the assignment immediately.
  </Step>
</Steps>

<Tip>
  If the outlet you need does not appear in the dropdown, it may not yet be created in the system. Navigate to **Customer Management** to add the outlet first, then return to complete the assignment.
</Tip>

## Filtering Terminals

The inventory list supports filtering and search to help you find specific terminals quickly. Use the search bar and filter controls above the table:

- **Terminal Serial Number** — enter a full or partial serial number to find a specific device.
- **MID** — filter by Merchant ID to find terminals belonging to a specific merchant agreement.
- **Customer Name** — filter by merchant customer to see all terminals deployed to a particular account.
- **Terminal Outlet** — filter by outlet name to narrow results to a specific merchant location.
- **Terminal Status** — filter by lifecycle state (INSTORE, ASSIGNED, INREPAIR, DECOMMISSION) to manage a specific subset of your estate.

You can combine multiple filters at the same time. Click **Reset** to clear all active filters and return to the full list.
