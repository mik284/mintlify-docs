Every transaction in Lango has a dedicated detail page that presents the complete payment record in one place. From the terminal and operator that processed the payment to the card details and response from the switch — all of it is available here without needing to query a separate system. You can also print a duplicate receipt directly from this page.

<Note>
  Access to transaction detail pages requires the **`canViewTransaction`** permission (`standard.transaction.view`). Some fields may not be visible depending on your role permissions and the transaction type.
</Note>

---

## Opening a Transaction Detail

From the [Transactions list](/transactions/viewing-transactions), locate the payment you want to inspect and click the <Icon icon="eye" /> **View** icon in the **Action** column at the far right of the row. Lango navigates you to the detail page at the route `/transactions/:id`, where `:id` is the unique identifier of that transaction record.

<Tip>
  Use your browser's **Back** button or the **Back** button displayed at the top-left of the detail page to return to the transaction list. Lango preserves your last filter state, so you return to the same filtered results without having to re-enter your search criteria.
</Tip>

## Payment Details

This section contains the financial and card-level data for the transaction.

| Field | Description |
|-------|-------------|
| **Amount** | The transaction amount, shown as a plain numeric value. The currency is displayed separately in the **Currency Name** and **Currency Symbol** fields. |
| **Currency Name** | The full name of the currency used for this transaction (for example, `Tanzanian Shilling`). |
| **Currency Symbol** | The currency symbol displayed alongside amounts throughout the portal (for example, `TZS`). |
| **Card Number (PAN)** | The masked Primary Account Number of the card used. Lango stores and displays the PAN in its masked form to comply with payment card data security requirements. |
| **Card Issuer** | The name of the institution that issued the card, if available from the switch response. |
| **Expiry Date** | The expiry date of the card as presented during the transaction. |
| **Entry Mode** | How the card data was captured — for example, chip, magnetic stripe, or contactless. |

## Terminal and Operator Information

This section tells you precisely which hardware processed the payment and who was operating it.

<CardGroup cols={2}>
  <Card title="Terminal ID (TID)" icon="tag" href="/transactions/viewing-transactions">

  </Card>

## Response Codes

Every transaction carries a **Response Code** returned by the payment switch, displayed alongside its human-readable message (for example, `00 : Approved` or `51 : Insufficient Funds`).

Response codes are standardised across ISO 8583-based payment networks and indicate the approval or decline reason at the issuer or switch level. Common codes you may encounter include:

| Code | Meaning |
|------|---------|
| `00` | Approved and completed successfully |
| `05` | Do not honour — generic decline from the issuer |
| `12` | Invalid transaction type |
| `14` | Invalid card number |
| `51` | Insufficient funds |
| `54` | Expired card |
| `55` | Incorrect PIN |
| `91` | Issuer or switch unavailable |

<Note>
  The full list of response codes in use depends on your payment switch configuration. Consult your switch operator or acquiring bank for a complete code reference specific to your network.
</Note>

## EMV and Technical Fields

For chip transactions, Lango also stores the raw EMV cryptographic values returned by the card. These fields are used by your payment switch team for dispute resolution and EMV compliance audits.

| Field | Description |
|-------|-------------|
| **Transaction Certificate (TC)** | The cryptogram generated by the chip card to confirm the transaction. |
| **Terminal Verification Results (TVR)** | A bitmap indicating the outcome of each EMV terminal check. |
| **Transaction Status Information (TSI)** | A bitmap indicating which functions were performed during the EMV processing flow. |
| **Cardholder Verification Method Results (CVMR)** | Indicates how the cardholder was verified (for example, PIN, signature, or no CVM required). |
| **Application Identifier (AID)** | Identifies the card application (for example, Visa Credit or Mastercard Debit) used during the transaction. |

## Printing a Receipt

A duplicate receipt is rendered directly on the transaction detail page. The receipt includes the merchant name, outlet address, TID, MID, RRN, PAN, card holder name, card issuer, transaction type, amount, and either the authorization code (for successful transactions) or the response code and decline message.

<Steps>
  <Step title="Locate the receipt preview">
    Scroll to the right-hand panel of the transaction detail page. The receipt is rendered in a standard till-receipt format and marked **DUPLICATE** as a watermark.
  </Step>
  <Step title="Click PRINT RECEIPT">
    Click the **PRINT RECEIPT** button above the receipt preview. Lango triggers the browser's native print dialogue.
  </Step>
  <Step title="Select your printer or save as PDF">
    Choose a connected printer from the print dialogue, or select **Save as PDF** to produce a digital copy for your records.
  </Step>
</Steps>

<Warning>
  Transaction records in Lango are **read-only**. You cannot edit, void, or reverse a transaction from this page. Reversals and adjustments must be initiated through your payment switch or acquiring system — not through Lango directly. Contact your switch operator if a transaction requires a reversal.
</Warning>
