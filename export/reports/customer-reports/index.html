    Customer Reports: Analyze Merchant Activity and Growth

    The Customer Reports section in Lango provides merchant onboarding summaries and activity data, exportable for portfolio analysis and compliance.

    The Customer Reports section in Lango gives bank operations and compliance teams a consolidated view of merchant portfolio activity. The Customer Activity report lists merchants onboarded within a chosen date range alongside their branch, terminals, and merchant IDs — giving you everything you need for portfolio reviews and regulatory submissions. Alongside it, the Users Created report tracks which portal user accounts were provisioned in a given period, serving as a key input for periodic access reviews and segregation-of-duties audits. Both reports support row selection and multi-format export.
    Access to customer reports requires the lite.reports.view permission. If the Reports menu is not visible in your sidebar, contact your system administrator.

    ​
    Customer Activity Report

    Navigate to Reports in the sidebar, then click Customers Report. The portal loads the report at /reports/customers-report.

    ​
    What the Report Shows

    The Customer Activity report returns one row per merchant matching your filters. Each row reflects the merchant’s onboarding details and their linked terminals:
    ColumnDescription
    Date RequestedThe timestamp when the merchant record was created in the system.
    Application TypeWhether this was a New merchant onboarding or a terminal Replacement for an existing merchant.
    Merchant NameThe registered business name of the merchant.
    BranchThe bank branch responsible for this merchant.
    RegionThe bank region associated with the merchant’s branch.
    Terminal Serial NumberAll terminal serial numbers linked to the merchant’s outlets, listed as a comma-separated string.
    MIDThe Merchant ID(s) assigned across the merchant’s device profiles, comma-separated.
    Created ByThe portal username of the operator who created the merchant record.
    Approved ByThe portal username of the operator who approved the merchant onboarding.

    ​
    Filtering the Customer Report

    Use the search panel to scope your results:
    • Date Range — Select a start and end date using the Date Requested filter to view only merchants onboarded in that window.
    • Application Type — Choose New Customer to see first-time onboardings, or Replacement to see existing merchants who received a new terminal.
    • Merchant Name — Enter a name to look up a specific merchant.
    • Terminal Serial Number — Filter to records associated with a specific device.
    Click Search to load the filtered results.
    Filter by Application Type: New Customer and export monthly to build a running record of portfolio growth for management reporting.

    ​
    Users Created Report

    Navigate to Reports in the sidebar, then click Users Created. The portal loads the report at /reports/users-created.

    ​
    What the Report Shows

    The Users Created report lists all portal user accounts registered in the system. It is particularly useful for access audits, enabling you to confirm that every active account is still required and that permissions remain appropriate.
    ColumnDescription
    Date CreatedThe timestamp when the user account was provisioned.
    UsernameThe login username assigned to the user.
    WorkgroupsThe workgroups the user belongs to, comma-separated.
    RolesThe roles assigned to the user across all workgroups, comma-separated and deduplicated. Each role links to its permission detail view.
    Created ByThe username of the operator who created the account.
    Approved ByThe username of the operator who approved account creation.
    Employee IDThe employee identifier associated with the user.
    Identification NumberThe national or institutional identification number on record.
    StatusActive or Inactive — whether the account is currently enabled.
    Last LoginThe timestamp of the user’s most recent login, or Never if the account has never been accessed.
    Run the Users Created report as part of a quarterly access review. Export it to Excel, then cross-reference the Status and Last Login columns to identify accounts that should be deactivated or have their roles reviewed.

    ​
    Exporting Customer and User Reports

    Both reports support row-level selection and three export formats. You can select specific rows before exporting to limit the output to those records, or leave all rows unchecked to export the full filtered result set.
    1

    Select the report

    Open either the Customers Report or the Users Created report from the Reports sidebar.
    2

    Apply filters (Customers Report only)

    For the Customers Report, set a date range and any additional filters, then click Search to load matching records. The Users Created report displays all user accounts automatically without a search filter.
    3

    Select rows (optional)

    Check individual rows if you want to export a subset of the results, or leave all rows unchecked to export everything in the current filtered view.
    4

    Choose your export format

    Click the appropriate button in the top-right toolbar:
    • Export to CSV — For loading into reconciliation or data tools.
    • Export to Excel — For review and annotation in Microsoft Excel (.xlsx).
    • Export to PDF — For formal submissions or printing (Customers Reports.pdf / Users Created Reports.pdf).
    5

    Download the file

    The browser downloads the file immediately with the relevant filename.

    ​
    Exported Columns — Customers Report

    Date Requested, Merchant Name, Branch, Region, Terminal Serial Number, Merchant ID, Created By, Approved By, Application Type.

    ​
    Exported Columns — Users Created Report

    Date Created, Username, Workgroups, Roles, Created By, Approved By, Employee ID, Identification Number, Status, Last Login.
