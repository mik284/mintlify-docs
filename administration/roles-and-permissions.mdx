Access control in Lango is built on a layered model that gives you fine-grained control over what every user in your organisation can see and do. Rather than assigning permissions directly to individuals, you compose permissions into roles, bundle roles into workgroups, and then assign users to workgroups. This approach makes it easy to onboard new staff, enforce consistent access policies, and adapt quickly when responsibilities change.

<Tip>
  Model your workgroups around your team structure — for example, **Terminal
  Operations**, **Compliance**, **Customer Onboarding**, and **Finance**. Each
  workgroup should reflect a distinct set of job responsibilities.
</Tip>

---

## Understanding the access model

Lango uses a four-tier hierarchy to manage access:

```
Permissions  →  Roles  →  Workgroups  →  Users
```

| Layer           | Description                                                                                      |
| --------------- | ------------------------------------------------------------------------------------------------ |
| **Permission**  | An atomic action, such as `lite.customer.create` or `lite.terminal.view`. Permissions are predefined by the system. |
| **Role**        | A named collection of permissions. For example, a "Customer Onboarding Officer" role might bundle `lite.customer.create`, `lite.customer.view`, and `lite.outlet.create`. |
| **Workgroup**   | A named collection of roles. A single user can inherit permissions from multiple roles through their workgroup. |
| **User**        | A portal account assigned to exactly one workgroup, which determines the complete set of actions that user is allowed to perform. |

This separation means you can change what an entire team can do by editing a single workgroup — no need to update every individual account.

<Note>
  Changes to a workgroup's roles take effect immediately for all users currently
  assigned to that workgroup.
</Note>

## Viewing roles

<Steps>
  <Step title="Navigate to Roles">
    Go to **User Management → Roles** in the left sidebar. The **List of Roles** table appears.
  </Step>
  <Step title="Browse the role list">
    The table shows each role's **Name**, **Description**, and **Date/Time Created**. Click the **View** button (eye icon) next to any role to see its full list of assigned permissions.
  </Step>
  <Step title="Search for a role">
    Use the search fields above the table to filter roles by name or description.
  </Step>
</Steps>

---

## Creating a role

<Steps>
  <Step title="Open the Roles page">
    Navigate to **User Management → Roles**.
  </Step>
  <Step title="Click Create Role">
    Click the **Create Role** button in the top-right toolbar. A modal form appears.
  </Step>
  <Step title="Enter the role details">
    Provide a **Role Name** and a **Role Description** that clearly reflects the job function this role supports (for example, "Terminal Technician" or "Compliance Reviewer").
  </Step>
  <Step title="Select permissions">
    Use the **Permission Cascader** to browse and select individual permissions. Permissions are organised by category (customer, terminal, transaction, user, settings, and so on). Select all the actions that users in this role need to perform.
  </Step>
  <Step title="Save the role">
    Click **Save**. The new role appears in the list immediately and is ready to be added to a workgroup.
  </Step>
</Steps>

---

## Creating a workgroup

<Steps>
  <Step title="Open the Workgroups page">
    Navigate to **User Management → Workgroups**.
  </Step>
  <Step title="Click Add Workgroup">
    Select **Add Workgroup** in the toolbar. A modal form appears.
  </Step>
  <Step title="Name the workgroup">
    Enter a **Workgroup Name** and a **Workgroup Description**. Use alphanumeric characters only — no special characters are permitted.
  </Step>
  <Step title="Assign roles">
    From the **Roles** dropdown, select one or more roles to include in this workgroup. Users assigned to this workgroup will inherit the combined permissions of all selected roles.
  </Step>
  <Step title="Save the workgroup">
    Click **Save**. The workgroup is created and can now be assigned to new or existing users.
  </Step>
</Steps>

## Permission reference

The permissions available in Lango are grouped into the following categories. Use the accordions below to find the specific permission strings you need when building roles.

<Accordion title="Customer permissions">
  These permissions control access to customer (merchant) records and their associated outlets and operators.

  | Permission key                  | What it allows                                      |
  | ------------------------------- | --------------------------------------------------- |
  | `lite.customer.view`            | View the customer list and individual customer records |
  | `lite.customer.create`          | Create new customer records                         |
  | `lite.customer.update`          | Edit existing customer records                      |
  | `lite.customer.delete`          | Delete or archive customer records                  |
  | `lite.outlet.view`              | View outlets belonging to a customer                |
  | `lite.outlet.create`            | Add new outlets to a customer                       |
  | `lite.outlet.update`            | Edit outlet details                                 |
  | `lite.outlet.delete`            | Remove outlets from a customer                      |
  | `lite.operator.view`            | View operators assigned to a terminal or outlet     |
  | `lite.operator.create`          | Create new operator accounts                        |
  | `lite.operator.update`          | Edit operator details                               |
  | `lite.operator.delete`          | Remove operators                                    |
  | `lite.operator.activate`        | Activate or deactivate an operator account          |
  | `lite.operator.reset`           | Reset an operator's PIN or password                 |
</Accordion>
